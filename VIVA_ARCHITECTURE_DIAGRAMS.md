## 🔑 Key Points to Explain

1. **Frontend** - React with Axios
2. **Backend** - Spring Boot with layered architecture (Controller → Service → Repository)
3. **Database** - MySQL with JPA/Hibernate ORM
4. **Communication** - RESTful API (HTTP/JSON)


## 🔐 GOOGLE OAUTH FLOW

```
┌──────────────┐
│  User on     │
│  Frontend    │
└────┬─────────┘
     │
     │ Clicks "Login with Google"
     │
     ▼
┌────────────────────────┐
│ Google OAuth Dialog    │
│ (Google handles auth)  │
└────┬───────────────────┘
     │
     │ User enters Google credentials
     │ (secure - on Google servers)
     │
     ▼
┌────────────────────────┐
│ Google generates JWT   │
│ (cryptographically     │
│  signed with Google's  │
│  private key)          │
└────┬───────────────────┘
     │
     │ Google returns token to frontend
     │ (cannot be forged - signed by Google)
     │
     ▼
┌────────────────────────┐
│ Frontend receives JWT  │
│ and sends to backend:  │
│                        │
│ POST /api/auth/google  │
│ {                      │
│   "token": "eyJhbG..." │
│ }                      │
└────┬───────────────────┘
     │
     ▼
┌────────────────────────┐
│ Spring Backend         │
│ receives token         │
└────┬───────────────────┘
     │
     │ Service layer: verifyGoogleToken(token)
     │
     ▼
┌────────────────────────┐
│ GoogleIdTokenVerifier  │
│                        │
│ 1. Check signature     │
│    (verify it's from   │
│    Google using their  │
│    public key)         │
│                        │
│ 2. Check expiration    │
│    (token not expired) │
│                        │
│ 3. Extract email       │
│    from payload        │
└────┬───────────────────┘
     │
     │ If verification FAILS:
     │ └─> Throw exception
     │     └─> Return 400 Bad Request
     │
     │ If verification SUCCEEDS:
     │
     ▼
┌────────────────────────┐
│ Check if user exists   │
│ in local database      │
│                        │
│ Query: SELECT * FROM   │
│ users WHERE email=?    │
└────┬───────────────────┘
     │
     ├─ User exists?
     │  └─> Return existing user
     │
     └─ User NOT exists?
        └─> Create new user
           └─> Set role = STUDENT
           └─> Save to DB
           └─> NO password stored
               (Google handles auth)
         │
         ▼
┌────────────────────────┐
│ Build AuthResponse:    │
│ {                      │
│   "id": 123,           │
│   "name": "John",      │
│   "email": "...",      │
│   "role": "STUDENT"    │
│ }                      │
│                        │
│ (no password returned) │
└────┬───────────────────┘
     │
     │ Return 200 OK
     │
     ▼
┌────────────────────────┐
│ Frontend:              │
│ - Store auth data      │
│ - Redirect to dashboard
│ - Show user profile    │
└────────────────────────┘
```

## 💾 DATA PERSISTENCE EXAMPLE

### User Registration Data Flow

```
Frontend React Form
├─ name: "John Doe"
├─ email: "john@example.com"
├─ password: "SecurePass123"
└─ role: [auto-set to STUDENT]
         │
         ▼
Axios POST to /api/auth/register
{
  "name": "John Doe",
  "email": "john@example.com",
  "password": "SecurePass123"
}
         │
         ▼
Spring UserController.register()
         │
         ▼
UserServiceImpl.register()
├─ Validate name not empty
├─ Validate email format & unique
├─ Validate password >= 6 chars
├─ Hash password: BCrypt.encode("SecurePass123")
│  → Returns: "$2a$10$..."
└─ Create User entity
         │
         ▼
User entity:
{
  id: null (will be auto-generated),
  name: "John Doe",
  email: "john@example.com",
  password: "$2a$10$...",  (HASHED)
  role: UserRole.STUDENT,
  blocked: false,
  loginCount: 0,
  createdAt: LocalDateTime.now()
}
         │
         ▼
UserRepository.save(user)
         │
         ▼
Hibernate converts entity to SQL:

  INSERT INTO users (
    id, name, email, password, role,
    blocked, login_count, created_at
  ) VALUES (
    NULL, 'John Doe', 'john@example.com',
    '$2a$10$...', 'STUDENT',
    false, 0, '2024-04-26 10:30:00'
  )
         │
         ▼
MySQL Database:

  ┌─────┬───────────┬────────────────────┬──────────┬────────┬─────────┬───────────┐
  │ id  │ name      │ email              │ password │ role   │ blocked │ createdAt │
  ├─────┼───────────┼────────────────────┼──────────┼────────┼─────────┼───────────┤
  │ 1   │ John Doe  │ john@example.com   │ $2a$... │ STUDENT│ false   │ 2024-...  │
  └─────┴───────────┴────────────────────┴──────────┴────────┴─────────┴───────────┘
         │
         ▼
JPA returns User object with generated ID = 1
         │
         ▼
Convert to AuthResponse DTO:
{
  "id": 1,
  "name": "John Doe",
  "email": "john@example.com",
  "role": "STUDENT"
  // NOTE: password NOT included
}
         │
         ▼
Wrap in ApiResponse:
{
  "success": true,
  "message": "User registered successfully.",
  "data": {
    "id": 1,
    "name": "John Doe",
    "email": "john@example.com",
    "role": "STUDENT"
  }
}
         │
         ▼
Return 200 OK to frontend
         │
         ▼
Frontend receives response
├─ Stores user data in localStorage
├─ Redirects to dashboard
└─ Shows "Welcome, John Doe!"
```
